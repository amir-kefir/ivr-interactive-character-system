### [ Сценарий 1 – Регистрация пользователя ]

1. Пользователь запускает приложение
2. Система отображает экран входа и регистрации
3. Пользователь выбирает создание аккаунта
4. Пользователь вводит логин
5. Пользователь вводит пароль
6. Пользователь подтверждает регистрацию
7. Система отправляет данные на backend
8. Сервер проверяет корректность введённых данных
9. Если данные корректны, сервер создаёт аккаунт
10. Система уведомляет пользователя об успешной регистрации
11. Запускается Сценарий 2 – Вход в систему

---

### [ Сценарий 2 – Вход в систему ]

1. Пользователь запускает приложение
2. Система отображает экран авторизации
3. Пользователь вводит логин
4. Пользователь вводит пароль
5. Пользователь нажимает кнопку входа
6. Система отправляет запрос на сервер
7. Сервер проверяет данные
8. Если данные верны, система открывает основное 3D-пространство
9. Пользователь получает доступ к функционалу приложения

---

### [ Сценарий 3 – Выбор виртуального персонажа ]

1. Пользователь входит в систему
2. Система отображает список доступных персонажей
3. Пользователь просматривает список
4. Пользователь выбирает персонажа
5. Система запрашивает параметры персонажа с сервера
6. Backend передаёт конфигурацию персонажа
7. Клиент загружает 3D-модель
8. Персонаж появляется в сцене
9. Пользователь получает возможность начать диалог

---

### [ Сценарий 4 – Начало диалога ]

1. Пользователь выбирает персонажа
2. Система отображает интерфейс общения
3. Пользователь вводит сообщение
4. Система отправляет сообщение на backend
5. Сервер анализирует текст
6. Сервер определяет эмоциональное состояние персонажа
7. Нейросеть генерирует ответ
8. Ответ передаётся в клиент
9. Персонаж воспроизводит реплику
10. Персонаж отображает соответствующую анимацию по результату 6 пункта

---

### [ Сценарий 5 – Завершение диалога ]

1. Пользователь ведёт диалог
2. Пользователь выбирает завершение общения
3. Система фиксирует окончание диалога
4. Система сохраняет историю общения 
5. Система скрывает интерфейс диалога
6. Пользователь возвращается в основное меню

---

### [ Сценарий 6 – Создание виртуального персонажа ]

1. Пользователь открывает меню создания персонажа
2. Система отображает доступные пресеты моделей
3. Пользователь выбирает 3D-модель
4. Пользователь выбирает файл текстуры на своём компьютере (.png/.jpg) через диалог выбора файла
5. Пользователь настраивает параметры характера
6. Пользователь вводит описание или предысторию персонажа
7. Данные отправляются на backend
8. Сервер сохраняет персонажа

---

### [ Сценарий 7 – Поиск персонажей ]

1. Пользователь открывает меню поиска
2. Система отображает доступные фильтры
3. Пользователь выбирает параметры фильтрации
4. Пользователь запускает поиск
5. Backend получает параметры запроса
6. Сервер выполняет фильтрацию
7. Сервер возвращает список персонажей
8. Система отображает результаты
9. Пользователь выбирает персонажа

---

### [ Сценарий 8 – Сохранение истории общения ]

1. Пользователь завершает разговор
2. Система формирует журнал диалога
3. Данные отправляются на backend
4. Сервер сохраняет историю общения
5. Пользователь открывает раздел истории
6. Система запрашивает данные с сервера
7. Backend передаёт сохранённые диалоги
8. Пользователь просматривает историю общения

---

### [ Сценарий 9 – Обработка чувствительного сообщения ]

1. Пользователь отправляет сообщение
2. Backend анализирует текст
3. Система выявляет чувствительный контекст
4. Активируются ограничения поведения персонажа
5. Нейросеть формирует корректный ответ
6. Ответ передаётся в клиент
7. Персонаж отображает изменённую реакцию
8. Пользователь получает безопасный ответ системы
